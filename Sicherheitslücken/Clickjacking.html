Clickjacking (UI Redressing)

Erkennung

Technisch gesehen kann jede Website, die in einen iFrame eingebettet werden kann, für Clickjacking-Angriffe anfällig sein. Deshalb ist es sowohl für Website-Administratoren als auch für Endnutzer so wichtig, proaktiv zu handeln, um sie zu verhindern.
Beispiel-Code (Clickjacking.html): <iframe src="https://www.onlineshop.example/account"></iframe>

Ursache

Die Ursache von Clickjacking ist, dass eine Webseite in einem unsichtbaren oder manipulierten iFrame (eine Art Fenster, das eine andere Webseite lädt) auf einer bösartigen Webseite eingebettet wird.
Dadurch können Nutzer unwissentlich auf etwas klicken, das sie nicht sehen, z. B. einen Knopf auf der echten Webseite. Der Angreifer nutzt das aus, um z. B. sensible Daten zu stehlen oder Aktionen auszuführen, die der Nutzer nicht beabsichtigt hat.

Behebung

Um Clickjacking zu verhindern, sollten iFrames blockiert werden. Dafür muss der Server so eingestellt werden, dass er den folgenden Header in die HTTP-Response einfügt:
X-Frame-Options: DENY
