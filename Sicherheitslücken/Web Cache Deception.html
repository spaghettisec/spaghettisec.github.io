Web Cache Deception

Erkennung

Ein Benutzer greift auf eine geschützte Seite zu, indem er sich einloggt. Der Angreifer fügt jedoch eine harmlose Endung wie „.jpg“ oder „.css“ zur URL hinzu. Der Webserver interpretiert dies oft als Anfrage für eine statische Ressource und speichert die Antwort im Cache. Der Angreifer kann dann den gecachten Inhalt abrufen und auf die sensiblen Daten zugreifen.

Ursache

Der Webserver speichert geschützte Inhalte im Cache, weil er nicht prüft, ob die Anfrage öffentlich oder sensibel ist. Oft passiert dies, wenn URLs mit Endungen wie „.jpg“ oder „.css“ fälschlicherweise als statische Dateien erkannt werden.

Behebung

Entwickler müssen sensible Endpunkte so konfigurieren, dass diese nicht vom CDN-Provider oder anderen Caching-Systemen gespeichert werden.
